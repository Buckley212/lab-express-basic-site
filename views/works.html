Overview

Providing feedback, like mark ups and commenting, is a key use case for collaboration, but this functionality was limited in Box. How could we increase engagement and promote the feedback loop in Box? I lead the redesign to extend our feature set to allow users to create annotations.

The Challenge

The lifecycle of a document begins with creation, then sharing with others to get feedback, then editing— iterating until the final draft and approval. Users could not complete this feedback loop in Box, and thus were leaving Box to complete their workflows. We were losing users

Creating an Annotation

The central feature of this project was allowing the user to create annotations. We landed on three annotation types that fit within our scope: point, highlight, and drawing. These supported different use cases, for example, creatives who wanted to give feedback on images, lawyers who wanted to annotate contracts, and teachers who wanted to mark up homework assignments

Threaded Conversations

A simple change that resulted in in streamlined work processes. Previously, users could not reply to any comment— they had to sort through the sidebar to track down the conversation. Threading comments together grouped conversations together for easy reading. I also updated the UX of the sidebar to show newest activity at the bottom, rather than the top

Viewing a Previous Version

One enormous challenge for me was overcoming technical constraints. One in particular resulted in all annotations on the file itself being lost with any new version upload. To combat this, I ensured there was a permanent record of every annotation in the sidebar, as well as a way to see all annotations on a previous version. Clicking on any annotation would jump the user to the exact version and location so they could compare changes and track the feedback
